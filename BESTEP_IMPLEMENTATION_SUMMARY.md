# BESTEP 整合實作摘要

## ✅ 已確認的業務規則

### 1. 報名成功定義
- `status='success'` 即為報名成功
- `status='approved'` 僅代表資料完整，不代表成功報名

### 2. 團體名次計算
- 依**團體聽、說、讀、寫分數平均排名**
- 並列規則：排名相同者視為並列同一名次
- 名次跳過規則：名次並列時，其後名次不再遞補，直接跳過並列名次數依序計算
- 獎勵金額：第1名5,000元 → 第2名4,000元 → ... → 第11-20名1,000元

### 3. 達標標準
- CEFR 等級 B2 以上為達標
- 判斷方式：依 `level` 欄位（B2, C1, C2 為達標）

### 4. 考試場次
- 一個學期僅一場 BESTEP 考試
- 包含 LRSW（聽讀說寫）四項測驗

### 5. 成績格式
- L/R/S/W 四個分數分開
- 總分自動計算（聽+讀+說+寫）

### 6. 班級定義
- 同一學生可以在同一學期屬於多個班級

## 📊 資料模型簡化

### 關鍵變更

1. **examSessionId → semester**
   - 由於一個學期一場，直接使用 `semester` 作為唯一識別
   - 簡化查詢邏輯

2. **移除 examType**
   - 固定為 LRSW，不需要區分

3. **成績表欄位**
   - 僅保留：`listeningScore`, `readingScore`, `speakingScore`, `writingScore`
   - `totalScore` 自動計算
   - 新增 `passed` 欄位（B2 以上）

4. **團體名次表**
   - 固定規則：平均分排序
   - 新增 `rewardAmount` 欄位
