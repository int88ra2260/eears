# 簽到功能檢查與修復報告

## 問題診斷結果

### 核心問題
**所有預約記錄都是「未簽到」狀態，導致班級統計資料為0**

### 詳細發現

1. **簽到狀態分布**
   - 總預約數：2388筆
   - 已簽到：0筆（0%）
   - 未簽到：2388筆（100%）

2. **簽到API限制**
   - 原限制：只能對當天的活動進行簽到
   - 影響：所有過去的活動無法補簽到
   - 結果：歷史活動的預約都保持「未簽到」狀態

3. **114-1學期活動情況**
   - 114-1學期範圍：2025-08-01 到 2026-01-31
   - 114-1學期活動數：93個
   - 但這些活動（8月之後）都沒有預約記錄
   - 所有預約都在更早的日期（2025-04-14 到 2025-11-04）

4. **學生ID匹配問題**
   - 班級學生：200人
   - 有預約的班級學生：僅58人（29%）
   - 142位班級學生沒有預約記錄（71%）

---

## 已實施的修復

### 1. 修改簽到API - 允許補簽到

**檔案：** `reservation-backend/routes/eventRouter.js`

**修改內容：**
- 允許管理員和執行長對過去的活動進行補簽到
- 一般用戶（工讀生、老師）仍只能對當天的活動進行簽到
- 補簽到時會記錄日誌

**修改後的邏輯：**
```javascript
// 檢查活動是否為當天（管理員和執行長可以對過去的活動進行補簽到）
const today = new Date().toISOString().split('T')[0];
const userRole = req.user.role;
const isAdmin = userRole === 'admin';
const isExecutive = userRole === 'teacher' && req.user.teacherLevel === 'executive';
const canBackdateCheckin = isAdmin || isExecutive;

if (event.date !== today && !canBackdateCheckin) {
  return res.status(400).json({ error: "只能對當天的活動進行簽到" });
}
```

### 2. 修改前端簽到功能

**檔案：** `reservation-frontend/src/components/AdminHome.js`

**修改內容：**
- 管理員和執行長可以看到「補簽到」按鈕（對過去的活動）
- 補簽到時會顯示確認對話框
- 一般用戶仍只能對當天的活動進行簽到

## 使用說明

### 如何進行補簽到

1. **登入管理員或執行長帳號**
2. **進入活動報表**
3. **選擇一個過去的活動**
4. **點擊「查看預約」**
5. **對未簽到的學生點擊「補簽到」按鈕**
6. **確認補簽到操作**

### 批量簽到（推薦）

如果有很多學生需要補簽到，可以使用「匯入刷卡Excel」功能：

1. **進入活動報表**
2. **選擇活動並點擊「查看預約」**
3. **點擊「匯入刷卡Excel」按鈕**
4. **上傳包含刷卡時間的Excel檔案**
5. **系統會自動批量簽到**

## 後續建議

### 方案 A：手動補簽到（適用於少量學生）

1. 使用管理員帳號登入
2. 逐個活動進行補簽到
3. 適合：學生數量較少的情況

### 方案 B：批量匯入簽到（推薦，適用於大量學生）

1. 準備包含刷卡時間的Excel檔案
2. 使用「匯入刷卡Excel」功能批量簽到
3. 適合：有刷卡記錄的情況

### 方案 C：修改統計邏輯（臨時方案）

如果暫時無法補簽到，可以修改統計邏輯，將「有預約」也算作參與：

**修改檔案：** `reservation-backend/controllers/adminClassesController.js`

將查詢條件從：
```sql
WHERE r.checkinStatus = '已簽到'
```

改為：
```sql
WHERE r.studentId IN (:studentIds)
  AND e.date BETWEEN :startDate AND :endDate
```

這樣會計算所有預約（不要求簽到），但會失去簽到統計的準確性。

## 注意事項

1. **補簽到功能僅限管理員和執行長使用**
2. **補簽到會記錄實際的簽到時間（當前時間），不是活動當天的時間**
