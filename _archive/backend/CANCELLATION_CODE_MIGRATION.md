# 取消預約驗證碼功能 - 資料庫遷移說明

## 概述

此功能新增了取消預約驗證碼機制，需要更新資料庫結構。

## 資料庫變更

### 新增欄位

在 `reservations` 資料表中新增 `cancellationCode` 欄位：

```sql
ALTER TABLE reservations 
ADD COLUMN cancellationCode VARCHAR(255) NULL 
COMMENT '取消預約驗證碼' 
AFTER `group`;
```

## 功能說明

### 1. 建立預約時
- 系統會自動生成 6 位數字的驗證碼
- 驗證碼會儲存在資料庫中
- 驗證碼會隨預約成功郵件一併寄送給學生

### 2. 取消預約時
- **一般學生**：需要輸入驗證碼才能取消預約
- **管理員/執行長**：可以繞過驗證碼驗證（透過後台管理）

### 3. 安全性
- 驗證碼為 6 位數字，隨機生成
- 只有擁有驗證碼的人才能取消對應的預約
- 有效防止其他同學惡意取消預約

## 測試步驟

1. 建立一筆新預約
2. 檢查預約成功郵件是否包含驗證碼
3. 嘗試使用錯誤的驗證碼取消預約（應該失敗）
4. 使用正確的驗證碼取消預約（應該成功）
5. 測試管理員取消預約（應該不需要驗證碼）

## 注意事項

- 舊的預約紀錄不會有驗證碼（`cancellationCode` 為 `NULL`）
- 對於沒有驗證碼的舊預約，管理員仍可正常取消
- 一般學生無法取消沒有驗證碼的舊預約（需要聯繫管理員）
