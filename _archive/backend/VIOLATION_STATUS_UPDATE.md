# 違規狀態更新功能

## 概述
在預約詳情內進行活動結束檢查後，將未簽到的學生"狀態"欄改成"已登記違規"。

## 修改內容

### 1. 資料庫模型更新
- **檔案**: `models/Reservation.js`
- **修改**: 在 `checkinStatus` ENUM 中新增 "已登記違規" 選項
- **變更**: `('未簽到', '已簽到')` → `('未簽到', '已簽到', '已登記違規')`

### 2. 資料庫遷移
- **檔案**: `migrations/add-violation-status-to-reservations.js`
- **功能**: 更新現有資料庫的 ENUM 欄位，新增 "已登記違規" 狀態

### 3. 後端 API 更新
- **檔案**: `routes/eventRouter.js`
- **API**: `POST /api/events/:id/auto-check`
- **修改**: 在處理未簽到學生時，將預約記錄的 `checkinStatus` 更新為 "已登記違規"

```javascript
// 更新預約記錄狀態為"已登記違規"
reservation.checkinStatus = '已登記違規';
await reservation.save();
```

### 4. 前端顯示更新
- **檔案**: `src/components/AdminPage.js`
- **修改內容**:
  - 簽到狀態顯示：新增紅色徽章顯示 "已登記違規" 狀態
  - 操作按鈕：對於 "已登記違規" 狀態的學生，隱藏違規按鈕，顯示狀態文字
  - 統計顯示：新增 "已登記違規" 人數統計

## 功能流程

1. **活動結束檢查**: 管理員點擊 "活動結束檢查" 按鈕
2. **自動處理**: 系統檢查所有預約學生的簽到狀態
3. **狀態更新**: 對於未簽到的學生：
   - 建立違規記錄到黑名單系統
   - 更新預約記錄狀態為 "已登記違規"
   - 更新使用者違規次數
4. **前端顯示**: 重新載入預約詳情，顯示更新後的狀態

## 視覺效果

- **已簽到**: 綠色徽章
- **未簽到**: 黃色徽章  
- **已登記違規**: 紅色徽章

## 注意事項

1. 需要執行資料庫遷移來更新現有資料
2. "已登記違規" 狀態的學生無法再次進行違規登記
3. 統計區域會顯示各狀態的人數統計
