# 報名成功拖曳排序功能說明

## 功能概述

在「報名成功」狀態的列表中，可以使用拖曳方式調整記錄的順序，系統會自動更新 `successSequence` 並重新排序。

## 功能位置

### 1. 拖曳排序功能

**位置**：培力英檢報名管理頁面 → 個人報名 → 報名成功標籤

**使用方式**：
1. 切換到「報名成功」狀態標籤
2. 在表格最左側會顯示拖曳手柄圖示（⋮⋮）
3. 將滑鼠移到拖曳手柄上，游標會變成「抓取」圖示
4. 按住滑鼠左鍵並拖曳到目標位置
5. 放開滑鼠後，系統會自動調整順序並更新 `successSequence`
6. 所有報名成功記錄會重新排序，確保序號連續（1, 2, 3...）

**視覺回饋**：
- 拖曳時：行會變成半透明（50% 透明度），並顯示藍色虛線邊框
- 游標變化：`grab` → `grabbing`

### 2. 調整順序按鈕（詳細資料 Modal）

**位置**：報名詳細資料 Modal（點擊表格中的記錄開啟）

**使用方式**：
- 上移按鈕（↑）：與前一位交換順序
- 下移按鈕（↓）：與後一位交換順序
- 調整後會立即生效並重新排序

## 技術實作

### 1. 調整順序按鈕的運作方式

**問題**：調整順序按鈕是否會改變 `successSequence`？

**答案**：**會改變**

**運作流程**：
1. 點擊上移/下移按鈕
2. 系統找到目標位置的記錄
3. **交換兩個記錄的 `successSequence`**
4. 調用 `reorderSuccessSequences()` 重新排序所有記錄
5. 所有記錄的 `successSequence` 會重新分配為連續的 1, 2, 3...

**API 端點**：
```
POST /api/english-test/registrations/:id/adjust-sequence
Body: {
  action: 'up' | 'down' | 'move',
  targetSequence?: number
}
```

### 2. 拖曳排序功能

**使用的套件**：`@dnd-kit/core`, `@dnd-kit/sortable`, `@dnd-kit/utilities`

**實作方式**：
- 使用 `DndContext` 包裹表格
- 使用 `SortableContext` 和 `SortableTableRow` 實現拖曳
- 拖曳結束時調用 `onDragEnd` 處理函數
- 通過 `handleAdjustSequence` API 更新順序

**啟用條件**：
- 僅在 `statusFilter === 'success'` 時啟用
- 僅在桌面版表格中啟用（手機版使用卡片佈局，不支援拖曳）

**拖曳手柄**：
- 位置：表格最左側（選擇框之前）
- 圖示：`fas fa-grip-vertical`（垂直三點）
- 樣式：灰色圖示，游標變為 `grab`

## 安裝依賴

請執行以下命令安裝必要的套件：

```bash
cd reservation-frontend
npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
```

## 使用限制

1. **僅報名成功狀態**：只有「報名成功」狀態的記錄可以調整順序
2. **桌面版專用**：拖曳功能僅在桌面版表格中可用，手機版使用卡片佈局
3. **拖曳距離**：需要拖曳 8px 以上才會啟動，避免與點擊衝突
4. **自動重新排序**：調整後會自動重新排序所有記錄，確保序號連續

## 注意事項

1. **序號唯一性**：每個報名成功記錄都有唯一的 `successSequence`
2. **序號連續性**：重新排序後序號會自動調整為連續的 1, 2, 3...
3. **效能考量**：重新排序會更新所有報名成功記錄，建議在非高峰時段進行大量調整
4. **資料一致性**：調整順序後，匯出 Excel 和照片 ZIP 都會使用更新後的 `successSequence`

## 常見問題

**Q: 為什麼拖曳沒有反應？**
A: 請確認：
1. 當前狀態標籤是否為「報名成功」
2. 是否在桌面版（非手機版）
3. 是否按住拖曳手柄（最左側的圖示）

**Q: 拖曳後序號會改變嗎？**
A: 會。拖曳後系統會：
1. 交換兩個記錄的 `successSequence`
2. 重新排序所有記錄
3. 重新分配連續序號（1, 2, 3...）

**Q: 可以一次調整多個記錄嗎？**
A: 目前僅支援單筆調整（拖曳或按鈕）。如需大量調整，建議分批進行。

**Q: 拖曳和按鈕調整有什麼區別？**
A: 
- **拖曳**：可以一次移動到任意位置，更直觀
- **按鈕**：只能上移或下移一位，適合微調

兩者都會更新 `successSequence` 並重新排序。
