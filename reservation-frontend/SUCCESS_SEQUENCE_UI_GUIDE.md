# 報名成功序號調整功能使用說明

## 功能概述

報名成功狀態的學生會根據變為「已通過」的時間順序自動分配序號（1, 2, 3...），並提供手動調整順序的功能。

## 功能位置

### 1. 匯出證件照按鈕

**位置**：培力英檢報名管理頁面 → 操作區（右上角）

**使用方式**：
1. 在「匯出篩選狀態」下拉選單中選擇「報名成功」
2. 點擊「匯出證件照」按鈕
3. 系統會下載 ZIP 檔案，檔名格式：`培力英檢報名成功證件照_YYYY-MM-DD.zip`
4. 照片檔名格式：`{序號}-{身分證字號}-{中文姓名}.{副檔名}`

**注意**：
- 只有「已通過」和「報名成功」狀態可以匯出證件照
- 照片會按照 `successSequence` 排序

### 2. 調整順序功能

**位置**：報名詳細資料 Modal（點擊表格中的記錄開啟）

**使用方式**：
1. 開啟報名詳細資料 Modal
2. 確認該記錄的狀態為「報名成功」
3. 在 Modal 標題欄右側會顯示：
   - **序號標籤**：顯示當前的 `successSequence`（例如：序號：123）
   - **上移按鈕**（↑）：點擊後與前一位交換順序
   - **下移按鈕**（↓）：點擊後與後一位交換順序
4. 調整後系統會自動重新排序所有報名成功記錄，確保序號連續（1, 2, 3...）

**操作限制**：
- 只有「報名成功」狀態的記錄可以調整順序
- 第一個記錄無法上移
- 最後一個記錄無法下移
- 調整後會立即生效並重新排序

### 3. 序號顯示

**在詳細資料 Modal 中**：
- 標題欄右側會顯示「序號：{數字}」標籤（僅限報名成功狀態）
- 此序號用於匯出 Excel 和照片時的編號

**在匯出的 Excel 中**：
- 第一欄「序號」會顯示 `successSequence`
- 報名成功狀態的記錄會按照 `successSequence` 排序

**在匯出的照片 ZIP 中**：
- 檔名會包含序號：`{序號}-{身分證字號}-{中文姓名}.{副檔名}`
- 例如：`123-A123456789-王小明.jpg`

## 自動排序邏輯

### 預設排序規則

當記錄變為「報名成功」時，會根據以下規則自動分配序號：

1. **主要排序**：`approvedAt` ASC（變為已通過的時間）
2. **次要排序**：`createdAt` ASC（報名時間，當 `approvedAt` 為 null 時使用）
3. **第三排序**：`id` ASC（報名編號）

### 重新排序時機

系統會在以下情況自動重新排序所有報名成功記錄：

1. **變為報名成功時**：新記錄會根據排序規則插入適當位置
2. **從報名成功變為其他狀態時**：移除該記錄後重新排序
3. **手動調整順序後**：交換順序後重新排序確保連續

## API 端點

### 調整順序
```
POST /api/english-test/registrations/:id/adjust-sequence
Body: {
  action: 'up' | 'down' | 'move',
  targetSequence?: number  // 僅用於 move 操作
}
```

### 重新排序所有記錄
```
POST /api/english-test/registrations/reorder-success
```

### 匯出證件照
```
GET /api/english-test/registrations/export/photos?status=success
```

## 注意事項

1. **序號唯一性**：每個報名成功記錄都有唯一的 `successSequence`
2. **序號連續性**：重新排序後序號會自動調整為連續的 1, 2, 3...
3. **已通過狀態**：已通過狀態不分配序號，匯出時使用報名編號 (`id`)
4. **批量操作**：批量變更狀態時會自動處理序號分配和重新排序
5. **效能考量**：重新排序會更新所有報名成功記錄，建議在非高峰時段進行大量調整

## 常見問題

**Q: 為什麼看不到調整順序的按鈕？**
A: 只有「報名成功」狀態的記錄才會顯示調整順序按鈕。請確認記錄狀態。

**Q: 調整順序後序號會改變嗎？**
A: 會。調整後系統會自動重新排序，確保所有序號連續（1, 2, 3...）。

**Q: 可以一次調整多個記錄的順序嗎？**
A: 目前僅支援單筆調整（上移、下移）。如需大量調整，建議使用「重新排序」API。

**Q: 匯出的照片檔名中的序號是什麼？**
A: 是 `successSequence`，與 Excel 匯出中的序號一致。
