## 1. 出席資料匯入格式

### 1.1 Excel 檔案格式

**重要**: 出缺席資料分為兩個檔案，需分別匯入：
- `培力英檢LR出缺席紀錄.xlsx` - LR（聽讀）場次
- `培力英檢SW出缺席紀錄.xlsx` - SW（說寫）場次

**工作表名稱**: 第一個工作表（系統會自動讀取）

**欄位順序**（第1行為標題列）：

| 欄位名稱 | 欄位代碼 | 資料類型 | 必填 | 說明 |
|---------|---------|---------|------|------|
| 學號 | studentId | 文字 | ✅ | 學生學號（格式：B/M/D/I/J + 9位數字） |
| 姓名 | name | 文字 | ✅ | 學生姓名 |
| 出席狀態 | attended | 文字 | ✅ | 「出席」或「缺席」（不區分大小寫） |
| 缺席原因 | absentReason | 文字 | ❌ | 若缺席，填寫原因（選填） |

### 1.2 資料範例

```
學號        姓名    出席狀態  缺席原因
B123456789  王小明  出席      
B987654321  李小華  缺席      請假
B111222333  張三    出席      
```

### 1.3 支援的格式變體

系統會自動識別以下欄位名稱變體：

**學號欄位**:
- `學號`、`Student ID`、`studentId`、`學號代碼`

**姓名欄位**:
- `姓名`、`Name`、`name`、`學生姓名`

**出席狀態欄位**:
- `出席狀態`、`Attendance`、`attended`、`是否出席`、`出席/缺席`

**缺席原因欄位**:
- `缺席原因`、`Absent Reason`、`absentReason`、`原因`、`備註`

**出席狀態值**:
- `出席`、`是`、`Y`、`Yes`、`true`、`1` → 視為出席
- `缺席`、`否`、`N`、`No`、`false`、`0` → 視為缺席

## 2. 成績資料匯入格式

### 2.1 Excel 檔案格式

**檔案名稱**: `培力英檢成績資料.xls`（試務官方提供）

**工作表名稱**: 第一個工作表（系統會自動讀取）

**欄位順序**（第1行為標題列）：

| 欄位名稱 | 欄位代碼 | 資料類型 | 必填 | 說明 |
|---------|---------|---------|------|------|
| 學號 | studentId | 文字 | ✅ | 學生學號（格式：B/M/D/I/J + 9位數字） |
| 姓名 | name | 文字 | ✅ | 學生姓名 |
| 聽力分數 | listeningScore | 數字 | ✅ | 聽力測驗分數 |
| 閱讀分數 | readingScore | 數字 | ✅ | 閱讀測驗分數 |
| 口說分數 | speakingScore | 數字 | ✅ | 口說測驗分數 |
| 寫作分數 | writingScore | 數字 | ✅ | 寫作測驗分數 |
| 聽力等級 | listeningLevel | 文字 | ✅ | 聽力 CEFR 等級（如：A1, A2, B1, B2, C1, C2） |
| 閱讀等級 | readingLevel | 文字 | ✅ | 閱讀 CEFR 等級 |
| 口說等級 | speakingLevel | 文字 | ✅ | 口說 CEFR 等級 |
| 寫作等級 | writingLevel | 文字 | ✅ | 寫作 CEFR 等級 |
| 總分 | totalScore | 數字 | ❌ | 總分（若未提供，系統自動計算：聽+讀+說+寫） |

### 2.2 資料範例

```
學號        姓名    聽力分數  閱讀分數  口說分數  寫作分數  聽力等級  閱讀等級  口說等級  寫作等級  總分
B123456789  王小明  90        85        88        87        B2        B2        C1        B2       350
B987654321  李小華  75        80        82        78        B1        B2        B2        B1       315
B111222333  張三    95        92        90        91        C1        C1        C1        C1       368
```

**注意**: 
- 成績資料包含 LR（聽讀）和 SW（說寫）兩場的成績
- 各項都有獨立的 CEFR 等級
- 達標判斷：各項都達 B2 以上才算達標

### 2.3 支援的格式變體

系統會自動識別以下欄位名稱變體：

**學號欄位**:
- `學號`、`Student ID`、`studentId`、`學號代碼`

**姓名欄位**:
- `姓名`、`Name`、`name`、`學生姓名`

**聽力分數欄位**:
- `聽力分數`、`Listening`、`listeningScore`、`聽力`、`L`、`聽力成績`

**閱讀分數欄位**:
- `閱讀分數`、`Reading`、`readingScore`、`閱讀`、`R`、`閱讀成績`

**口說分數欄位**:
- `口說分數`、`Speaking`、`speakingScore`、`口說`、`S`、`口說成績`

**寫作分數欄位**:
- `寫作分數`、`Writing`、`writingScore`、`寫作`、`W`、`寫作成績`

**聽力等級欄位**:
- `聽力等級`、`Listening Level`、`listeningLevel`、`聽力CEFR`、`聽力級別`

**閱讀等級欄位**:
- `閱讀等級`、`Reading Level`、`readingLevel`、`閱讀CEFR`、`閱讀級別`

**口說等級欄位**:
- `口說等級`、`Speaking Level`、`speakingLevel`、`口說CEFR`、`口說級別`

**寫作等級欄位**:
- `寫作等級`、`Writing Level`、`writingLevel`、`寫作CEFR`、`寫作級別`

**總分欄位**:
- `總分`、`Total`、`totalScore`、`總成績`、`合計`

## 3. 欄位說明

### 3.1 學號格式

- **格式**: `[B|M|D|I|J][0-9]{9}`
- **範例**: `B123456789`, `M987654321`
- **驗證**: 系統會驗證學號是否存在於 `english_test_registrations` 表中（且 `status='success'`）

### 3.2 分數範圍

- **預設範圍**: 0-150（可配置）
- **小數點**: 支援小數點後2位（如：87.5）
- **驗證**: 系統會驗證分數是否在合理範圍內

### 3.3 等級格式

- **格式**: `[A1|A2|B1|B2|C1|C2]`
- **範例**: `B2`, `C1`
- **達標標準**: `B2`, `C1`, `C2` 視為達標

## 4. 資料驗證規則

### 4.1 出席資料驗證

1. **學號驗證**:
   - ✅ 格式正確（B/M/D/I/J + 9位數字）
   - ✅ 存在於 `english_test_registrations` 表中
   - ✅ 報名狀態為 `status='success'`

2. **出席狀態驗證**:
   - ✅ 值為「出席」或「缺席」（不區分大小寫）
   - ✅ 若為「缺席」，建議填寫缺席原因（選填）

3. **重複資料處理**:
   - ✅ 若同一學號在同一學期已有出席記錄，則更新現有記錄
   - ✅ 更新時保留原有的 `importedAt` 和 `sourceFile`，更新 `updatedAt`

### 4.2 成績資料驗證

1. **學號驗證**:
   - ✅ 格式正確
   - ✅ 存在於 `english_test_registrations` 表中
   - ✅ 報名狀態為 `status='success'`

2. **分數驗證**:
   - ✅ 聽力、閱讀、口說、寫作分數皆為數字
   - ✅ 分數範圍在 0-150 之間（可配置）
   - ✅ 若提供總分，需驗證總分 = 聽力 + 閱讀 + 口說 + 寫作（允許 ±1 的誤差）

3. **等級驗證**:
   - ✅ 各項等級（聽力、閱讀、口說、寫作）需為有效的 CEFR 等級（A1, A2, B1, B2, C1, C2）
   - ✅ 達標判斷：各項都達 B2 以上才算達標（`passed = true`）
   - ✅ 整體等級：取最低項等級（如：聽力B2、閱讀B2、口說C1、寫作B1 → 整體B1）

4. **重複資料處理**:
   - ✅ 若同一學號在同一學期已有成績記錄，則更新現有記錄
   - ✅ 更新時保留原有的 `importedAt` 和 `sourceFile`，更新 `updatedAt`

## 5. 錯誤處理

### 5.1 錯誤類型

| 錯誤類型 | 錯誤代碼 | 說明 | 處理方式 |
|---------|---------|------|---------|
| 學號格式錯誤 | `INVALID_STUDENT_ID` | 學號格式不符合規範 | 跳過該筆，記錄錯誤 |
| 學號不存在 | `STUDENT_NOT_FOUND` | 學號不存在於報名記錄中 | 跳過該筆，記錄錯誤 |
| 報名狀態不符 | `REGISTRATION_NOT_SUCCESS` | 報名狀態不是 'success' | 跳過該筆，記錄錯誤 |
| 分數格式錯誤 | `INVALID_SCORE` | 分數不是數字或超出範圍 | 跳過該筆，記錄錯誤 |
| 總分不符 | `TOTAL_SCORE_MISMATCH` | 總分與各科分數總和不符 | 跳過該筆，記錄錯誤 |
| 等級格式錯誤 | `INVALID_LEVEL` | 等級格式不符合 CEFR 規範 | 跳過該筆，記錄錯誤 |
| 必填欄位缺失 | `MISSING_REQUIRED_FIELD` | 必填欄位為空 | 跳過該筆，記錄錯誤 |

### 5.2 錯誤報表

匯入完成後，若有多筆錯誤，系統會生成錯誤報表 Excel 檔案：

**錯誤報表格式**:

| 行號 | 學號 | 姓名 | 錯誤類型 | 錯誤訊息 | 原始資料 |
|------|------|------|---------|---------|---------|
| 3 | B999999999 | 測試 | STUDENT_NOT_FOUND | 找不到該學號的報名記錄 | ... |
| 5 | B111111111 | 測試2 | INVALID_SCORE | 聽力分數格式錯誤 | ... |

**錯誤報表下載**:
- 匯入 API 回傳 `errorFileUrl`
- 前端提供下載連結
- 檔案保留 7 天後自動刪除

## 7. 匯入 API 使用說明

### 7.1 出席資料匯入

**端點**: `POST /api/admin/bestep/attendance/import`

**請求格式**:
```
Content-Type: multipart/form-data

Body:
- file: File (Excel 檔案：培力英檢LR出缺席紀錄.xlsx 或 培力英檢SW出缺席紀錄.xlsx)
- semester: string (學期，如 '114-1')
- examType: string (考試類型：'LR' 或 'SW')
- examDate: string (考試日期，格式：YYYY-MM-DD)
```

**注意**: 需要分別匯入 LR 和 SW 兩場的出缺席資料

**回應格式**:
```json
{
  "success": true,
  "semester": "114-1",
  "examType": "LR",
  "examDate": "2025-01-15",
  "imported": 25,
  "skipped": 2,
  "errors": [
    {
      "row": 3,
      "studentId": "B999999999",
      "name": "測試",
      "error": "STUDENT_NOT_FOUND",
      "message": "找不到該學號的報名記錄"
    }
  ],
  "errorFileUrl": "/api/admin/bestep/attendance/import/errors/xxx.xlsx"
}
```

### 7.2 成績資料匯入

**端點**: `POST /api/admin/bestep/scores/import`

**請求格式**:
```
Content-Type: multipart/form-data

Body:
- file: File (Excel 檔案：培力英檢成績資料.xls)
- semester: string (學期，如 '114-1')
```

**注意**: 
- 成績資料包含 LR 和 SW 兩場的成績，一次匯入即可
- 系統會自動解析各項分數和等級

**回應格式**: 
```json
{
  "success": true,
  "semester": "114-1",
  "imported": 25,
  "skipped": 2,
  "errors": [
    {
      "row": 3,
      "studentId": "B999999999",
      "name": "測試",
      "error": "STUDENT_NOT_FOUND",
      "message": "找不到該學號的報名記錄"
    }
  ],
  "errorFileUrl": "/api/admin/bestep/scores/import/errors/xxx.xlsx"
}
```

## 8. 資料調整建議

### 8.1 若試務官方 Excel 格式不同

**方案A：提供格式轉換工具**
- 建立 Excel 範本檔案
- 提供欄位對應表
- 管理員手動調整後匯入

**方案B：支援多種格式自動識別**
- 系統自動識別欄位名稱變體（見上方「支援的格式變體」）
- 若欄位順序不同，系統自動對應

**方案C：提供格式轉換 API**
- 上傳原始 Excel
- 系統自動轉換為標準格式
- 下載轉換後的 Excel
- 確認後再匯入

### 8.2 建議流程

1. **第一次匯入前**:
   - ✅ 已收到試務官方 Excel 檔案：
     - `培力英檢成績資料.xls`
     - `培力英檢LR出缺席紀錄.xlsx`
     - `培力英檢SW出缺席紀錄.xlsx`
   - 確認欄位名稱和順序（系統會自動識別變體）
   - 若格式不同，提供欄位對應表或建立轉換規則

2. **匯入測試**:
   - 使用少量測試資料匯入
   - 檢查錯誤報表
   - 確認資料正確性
   - **特別注意**: 確認各項 CEFR 等級欄位名稱

3. **正式匯入**:
   - **步驟1**: 匯入 LR 出缺席資料（`examType='LR'`）
   - **步驟2**: 匯入 SW 出缺席資料（`examType='SW'`）
   - **步驟3**: 匯入成績資料（包含 LR 和 SW 的成績）
   - 檢查匯入結果
   - 下載錯誤報表（如有）
   - 修正錯誤後重新匯入

## 9. 注意事項

1. **學號格式**: 請確保學號格式正確，系統會嚴格驗證
2. **報名狀態**: 只有 `status='success'` 的報名記錄才能匯入出席/成績
3. **重複匯入**: 系統支援重複匯入（會更新現有記錄），但建議避免重複匯入
4. **資料備份**: 匯入前建議備份資料庫
5. **錯誤處理**: 匯入失敗的資料會記錄在錯誤報表中，請及時處理
