# 簽到與違規管理功能說明

## 功能概述

本系統新增了完整的簽到與違規管理功能，讓管理員和工讀生能夠在活動現場進行有效的學生管理。

## 新增功能

### 1. 簽到功能
- **支援手機操作**：工讀生或管理員可在活動現場使用手機直接幫學生簽到
- **即時狀態更新**：簽到狀態會即時顯示在預約清單中
- **簽到時間記錄**：系統會記錄每位學生的具體簽到時間

### 2. 違規登記功能
- **即時違規記錄**：管理員或工讀生可針對個別學生手動新增違規紀錄
- **違規類型分類**：支援「擾亂秩序」、「未遵守規定」、「其他」等類型
- **詳細描述**：可記錄違規的詳細描述
- **記錄者追蹤**：系統會記錄是誰登記的違規

### 3. 活動結束後自動檢查
- **自動違規檢查**：活動結束時自動檢查所有預約學生
- **未到記錄**：未簽到的學生會被自動記錄為「預約未到」
- **違規整合**：活動期間的違規記錄會自動整合到違規管理系統
- **黑名單機制**：達到違規次數的學生會自動進入黑名單

### 4. 預約詳情表頭更新
- **新表頭格式**：學號 | 姓名 | 組別 | 簽到狀態
- **組別自動分配**：系統會自動將學生平均分配到9個組別
- **狀態視覺化**：使用不同顏色的徽章顯示簽到狀態

## 資料庫變更

### 新增欄位
- `Reservations.checkinStatus`：簽到狀態（未簽到/已簽到）
- `Reservations.checkinTime`：簽到時間
- `Reservations.group`：學生分組

### 新增資料表
- `event_violations`：活動期間違規記錄表

## API 端點

### 簽到相關
- `POST /api/events/:id/checkin`：學生簽到
- `GET /api/events/:id/reservations`：取得預約詳情（包含簽到狀態）

### 違規管理
- `POST /api/events/:id/violations`：登記活動期間違規
- `GET /api/events/:id/violations`：取得活動違規記錄

### 自動檢查
- `POST /api/events/:id/auto-check`：執行活動結束後自動檢查

## 使用流程

### 活動進行中
1. 工讀生/管理員點擊「查看預約」開啟預約詳情
2. 學生到場時，點擊對應的「簽到」按鈕
3. 如有違規行為，點擊「違規」按鈕進行登記
4. 系統即時更新簽到狀態和違規記錄

### 活動結束後
1. 管理員點擊「活動結束檢查」按鈕
2. 系統自動檢查所有預約學生的簽到狀態
3. 未簽到學生自動記錄為「預約未到」
4. 活動期間違規記錄整合到違規管理系統
5. 達到違規次數的學生自動進入黑名單

## 權限控制

- **工讀生**：可進行簽到和違規登記
- **管理員**：擁有所有功能，包括活動結束後自動檢查

## 手機操作優化

- 響應式設計，支援手機螢幕
- 大按鈕設計，方便觸控操作
- 即時狀態更新，無需重新整理頁面

## 注意事項

1. 簽到功能僅在活動進行期間使用
2. 違規記錄會影響學生的黑名單狀態
3. 活動結束檢查功能僅限管理員使用
4. 所有操作都會記錄操作者資訊

## 技術實現

- 前端使用 React Bootstrap 組件
- 後端使用 Express.js 和 Sequelize ORM
- 資料庫支援 MySQL/PostgreSQL
- 即時狀態更新使用 React state 管理
