# 違規管理功能增強

## 概述
為了提升違規登記的效率，我們新增了多種更方便的處理方式，讓管理員可以更快速地處理學生違規事件。

## 新增功能

### 1. 批次違規登記
- **功能描述**: 支援一次輸入多個學生的違規資料
- **使用方式**: 
  - 在違規管理頁面點擊「批次登記」按鈕
  - 可以動態新增/刪除行數
  - 一次處理多筆違規資料
- **API端點**: `POST /api/blacklist/batchRecordViolations`

### 2. CSV檔案匯入
- **功能描述**: 支援上傳CSV檔案批次匯入違規資料
- **使用方式**:
  - 下載CSV範本檔案
  - 按照格式填寫違規資料 (學號,姓名,違規原因)
  - 上傳CSV檔案進行批次匯入
- **API端點**: 
  - `POST /api/blacklist/importCsv` - 匯入CSV檔案
  - `GET /api/blacklist/downloadTemplate` - 下載範本
- **CSV格式範例**:
  ```csv
  學號,姓名,違規原因
  B12345678,張三,無故未到
  B87654321,李四,中途離席
  ```

### 3. 快速違規登記
- **功能描述**: 從活動預約紀錄直接標記違規
- **使用方式**:
  - 在活動報表中點擊「查看預約」按鈕
  - 查看該活動的所有預約學生
  - 直接點擊「標記違規」按鈕進行違規登記
- **API端點**: `GET /api/events/:id/reservations`

### 4. 介面優化
- **模式切換**: 單筆登記 / 批次登記模式切換
- **視覺回饋**: 加入載入動畫和圖示
- **檔案狀態**: 顯示已選擇的檔案名稱
- **操作確認**: 改善使用者操作流程

## 技術實作

### 後端API
- 新增批次處理邏輯，支援部分成功/失敗回報
- 檔案上傳處理 (multer + csv-parser)
- 預約詳情查詢API

### 前端介面
- React Modal組件用於預約詳情顯示
- 動態表單處理批次輸入
- 檔案上傳和狀態管理

### 依賴套件
需要安裝以下套件 (如果尚未安裝):
```bash
npm install multer csv-parser dayjs
```

## 使用建議

1. **少量違規**: 使用單筆登記模式
2. **大量違規**: 使用批次登記或CSV匯入
3. **活動後處理**: 使用快速違規登記從預約紀錄標記
4. **定期處理**: 準備CSV檔案進行批次匯入

## 注意事項

- CSV檔案必須包含標題行
- 學號或姓名至少需要提供一個
- 違規原因為可選，預設為「違規」
- 系統會自動處理黑名單邏輯 (違規2次進入黑名單)
- 上傳的CSV檔案會在處理後自動刪除

## 錯誤處理

- 批次處理會回報成功/失敗筆數
- 失敗的記錄會記錄在console中
- 檔案格式錯誤會顯示相應提示
- 網路錯誤會顯示重試提示
