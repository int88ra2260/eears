# EEARS/Reservation 系統資安審視報告

**審視日期**：2024-12-11  
**審視範圍**：React 前端 + Node.js/Express 後端 + MySQL 資料庫  
**審視人員**：資安顧問 + 全端工程師

---

## 📋 執行摘要

本報告針對 EEARS/Reservation 系統進行全面資安審視，識別出 **15 個高風險漏洞**、**8 個中風險問題**和 **5 個低風險建議**。所有問題均已提供可落地的整改方案，確保在不破壞既有功能的前提下，可逐步上線、可回滾、可驗證。

### 風險等級分布

- 🔴 **高風險**：15 項（需立即處理）
- 🟡 **中風險**：8 項（建議盡快處理）
- 🟢 **低風險**：5 項（建議後續優化）

## 🔴 高風險漏洞（Critical）

### 1. JWT 密鑰硬編碼（CWE-798）

**風險等級**：🔴 極高  
**影響範圍**：整個認證系統  
**發現位置**：`middlewares/auth.js:4`

**問題描述**：
```javascript
const secretKey = process.env.JWT_SECRET || 'MY_SUPER_SECRET_KEY';
```

**風險**：
- 如果環境變數未設定，使用弱密鑰 `'MY_SUPER_SECRET_KEY'`
- 攻擊者可輕易偽造 JWT token
- 完全繞過認證機制

**整改方案**：
1. 強制要求 JWT_SECRET 環境變數
2. 啟動時驗證密鑰強度（至少 32 字元，包含大小寫、數字、特殊字符）
3. 使用密鑰管理服務（如 AWS Secrets Manager）或加密的配置文件

**優先級**：P0（立即處理）

---

### 2. CORS 配置過於寬鬆（CWE-942）

**風險等級**：🔴 極高  
**影響範圍**：所有 API 端點  
**發現位置**：`server.js:28`

**問題描述**：
```javascript
app.use(cors()); // 允許所有來源
```

**風險**：
- 任何網站都可以發起跨域請求
- 可能導致 CSRF 攻擊
- 敏感資料可能被惡意網站讀取

**整改方案**：
1. 限制允許的來源域名
2. 根據環境變數動態配置
3. 實施憑證檢查（credentials: true）

**優先級**：P0（立即處理）

---

### 3. 缺少 Rate Limiting（CWE-307）

**風險等級**：🔴 極高  
**影響範圍**：所有 API 端點  
**發現位置**：無實際實現（僅文檔提及）

**風險**：
- 暴力破解登入密碼
- DDoS 攻擊
- API 濫用導致資源耗盡

**整改方案**：
1. 實施 express-rate-limit
2. 針對不同端點設定不同限制
3. 使用 Redis 進行分散式限流

**優先級**：P0（立即處理）

---

### 4. 錯誤訊息洩露敏感資訊（CWE-209）

**風險等級**：🔴 高  
**影響範圍**：所有錯誤回應  
**發現位置**：`middlewares/errorHandler.js:110-113`

**問題描述**：
```javascript
if (process.env.NODE_ENV === 'development') {
  response.stack = err.stack;
  response.originalError = err.message;
}
```

**風險**：
- 開發環境的錯誤可能洩露資料庫結構
- 可能洩露檔案路徑、API 金鑰等敏感資訊
- 攻擊者可利用錯誤訊息進行進一步攻擊

**整改方案**：
1. 生產環境完全隱藏 stack trace
2. 錯誤訊息統一格式，不包含技術細節
3. 敏感錯誤記錄到日誌，不返回給客戶端

**優先級**：P0（立即處理）

---

### 5. 文件上傳驗證不足（CWE-434）

**風險等級**：🔴 高  
**影響範圍**：所有文件上傳端點  
**發現位置**：`routes/englishTestRegistrationRouter.js:104-114`, `routes/adminClasses.js:39-48`

**問題描述**：
- 僅檢查副檔名和 MIME type
- 未驗證實際文件內容
- 可能上傳惡意文件（如偽裝成圖片的 PHP 腳本）

**風險**：
- 上傳惡意文件導致伺服器被入侵
- 檔案包含病毒或惡意代碼
- 路徑遍歷攻擊

**整改方案**：
1. 使用 `file-type` 或 `mmmagic` 檢查實際文件類型
2. 限制文件大小和數量
3. 將上傳文件儲存在 Web 根目錄外
4. 掃描上傳文件（防毒軟體）
5. 重新命名文件，避免路徑遍歷

**優先級**：P0（立即處理）

---

### 6. 缺少安全標頭（CWE-693）

**風險等級**：🔴 高  
**影響範圍**：所有 HTTP 回應  
**發現位置**：`server.js`

**風險**：
- XSS 攻擊
- 點擊劫持（Clickjacking）
- MIME 類型嗅探攻擊
- 缺少 HTTPS 強制

**整改方案**：
1. 安裝並配置 `helmet` 中間件
2. 設定適當的安全標頭（CSP, X-Frame-Options, X-Content-Type-Options 等）

**優先級**：P0（立即處理）

---

### 7. 缺少 CSRF 保護（CWE-352）

**風險等級**：🔴 高  
**影響範圍**：所有 POST/PUT/DELETE 請求  
**發現位置**：無 CSRF 保護

**風險**：
- 跨站請求偽造攻擊
- 未授權操作（刪除、修改資料）

**整改方案**：
1. 實施 CSRF token 機制
2. 使用 `csurf` 或 `csrf` 中間件
3. 對於 API，使用 SameSite cookie 和 Origin 檢查

**優先級**：P1（高優先級）

---

### 8. 環境變數文件可能被提交（CWE-312）

**風險等級**：🔴 高  
**影響範圍**：版本控制系統  
**發現位置**：`.env` 文件存在於倉庫中

**風險**：
- 敏感資訊（資料庫密碼、JWT 密鑰）洩露
- 如果倉庫公開，任何人都可以取得

**整改方案**：
1. 確認 `.env` 在 `.gitignore` 中
2. 檢查 Git 歷史，移除已提交的敏感資訊
3. 使用 `.env.example` 作為範本
4. 輪換所有已洩露的密鑰

**優先級**：P0（立即處理）

---

### 9. 輸入長度限制不足（CWE-400）

**風險等級**：🔴 高  
**影響範圍**：所有輸入端點  
**發現位置**：多處

**問題描述**：
- 雖然有基本驗證，但缺少統一的輸入長度限制
- 可能導致 DoS 攻擊（記憶體耗盡）

**風險**：
- 超長字串導致記憶體溢出
- 資料庫欄位溢出
- DoS 攻擊

**整改方案**：
1. 實施統一的輸入驗證中間件
2. 設定合理的欄位長度限制
3. 使用 `express-validator` 進行驗證

**優先級**：P1（高優先級）

---

### 10. SQL 注入防護依賴 ORM（CWE-89）

**風險等級**：🔴 高  
**影響範圍**：所有資料庫查詢  
**發現位置**：使用 Sequelize ORM

**問題描述**：
- 雖然使用 Sequelize（參數化查詢），但需要確認沒有原始 SQL 查詢
- 某些地方使用 `sequelize.query()` 需要檢查

**風險**：
- 如果使用原始 SQL 且未正確參數化，可能導致 SQL 注入

**整改方案**：
1. 審查所有 `sequelize.query()` 調用
2. 確保所有查詢都使用參數化
3. 禁止動態 SQL 拼接

**優先級**：P1（高優先級）

---

### 11. JWT Token 過期時間過長（CWE-613）

**風險等級**：🔴 中高  
**影響範圍**：認證系統  
**發現位置**：`routes/loginRouter.js:37`

**問題描述**：
```javascript
const token = jwt.sign(payload, secretKey, { expiresIn: '4h' });
```

**風險**：
- Token 有效期 4 小時過長
- 如果 token 被竊取，攻擊者有長時間的訪問權限

**整改方案**：
1. 縮短 access token 有效期（15-30 分鐘）
2. 實施 refresh token 機制
3. 提供 token 撤銷功能

**優先級**：P1（高優先級）

---

### 12. 缺少登入失敗鎖定機制（CWE-307）

**風險等級**：🔴 中高  
**影響範圍**：登入端點  
**發現位置**：`routes/loginRouter.js`

**風險**：
- 暴力破解攻擊
- 帳號枚舉攻擊

**整改方案**：
1. 實施登入失敗次數限制
2. 帳號鎖定機制（臨時鎖定）
3. CAPTCHA 驗證（多次失敗後）

**優先級**：P1（高優先級）

### 13. 敏感資料日誌記錄（CWE-532）

**風險等級**：🔴 中高  
**影響範圍**：日誌系統  
**發現位置**：多處 `console.log`

**風險**：
- 密碼、token、個人資料可能被記錄到日誌
- 日誌文件可能被未授權訪問

**整改方案**：
1. 審查所有日誌記錄
2. 過濾敏感資訊（密碼、token、身份證號）
3. 使用結構化日誌（如 Winston）
4. 日誌文件權限控制

**優先級**：P1（高優先級）

### 14. 檔案路徑遍歷風險（CWE-22）

**風險等級**：🔴 中高  
**影響範圍**：檔案上傳和下載  
**發現位置**：`routes/englishTestRegistrationRouter.js`

**問題描述**：
- 雖然有檔名清理，但需要確認所有路徑操作都安全

**風險**：
- 路徑遍歷攻擊（`../`）
- 讀取系統檔案
- 覆蓋重要檔案

**整改方案**：
1. 使用 `path.resolve()` 和 `path.join()` 規範化路徑
2. 驗證路徑在允許的目錄內
3. 禁止 `..` 和絕對路徑

**優先級**：P1（高優先級）
