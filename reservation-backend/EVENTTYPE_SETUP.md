# 活動類型預約時間設定說明

## 概述
此更新為不同活動類型設定了不同的預約開始時間，以滿足不同活動的預約需求。

## 活動類型與預約時間規則

1. **English Table**
   - 預約開始時間：活動開始前一天的 00:00
   - 預約結束時間：活動開始前 2 小時
   - 保持原有邏輯不變

2. **Job Talk**
   - 預約開始時間：活動開始前一個禮拜的同一個weekday的中午 12:00
   - 預約結束時間：活動開始前 2 小時
   - 範例：9/23(一)的Job Talk → 9/16(一) 12:00開始預約；9/25(三)的Job Talk → 9/18(三) 12:00開始預約

3. **English Club**
   - 預約開始時間：上禮拜三的中午 12:00
   - 預約結束時間：活動開始前 2 小時

4. **International Forum**
   - 預約開始時間：上禮拜五的中午 12:00
   - 預約結束時間：活動開始前 2 小時

## 實作變更

### 後端變更
1. **Event 模型** (`models/Event.js`)
   - 新增 `eventType` 欄位，預設值為 'English Table'

2. **預約時間計算工具** (`utils/reservationTime.js`)
   - 新增 `calculateReservationTime()` 函式
   - 根據活動類型計算不同的預約開放時間

3. **預約路由** (`routes/reservationRouter.js`)
   - 使用新的時間計算邏輯

4. **活動路由** (`routes/eventRouter.js`)
   - API 回應中包含 `eventType` 欄位
   - 新增活動時支援 `eventType` 參數

### 前端變更
1. **預約時間計算工具** (`src/utils/reservationTime.js`)
   - 前端版本的時間計算邏輯

2. **活動列表組件** (`src/components/EventList.js`)
   - 使用新的時間計算邏輯顯示正確的預約開放時間

3. **管理員頁面** (`src/components/AdminPage.js`)
   - 新增活動表單包含活動類型選擇
   - 編輯活動表單包含活動類型選擇
   - 活動列表顯示活動類型

## 資料庫遷移

執行以下 SQL 腳本來更新現有資料庫：

```sql
-- 執行 migrations/add-eventtype-to-events.sql
```

此腳本會：
1. 為 Events 表新增 `eventType` 欄位
2. 根據活動名稱為現有活動設定適當的類型
3. 預設值設為 'English Table'

## 使用方式

### 管理員
1. 登入後台管理系統
2. 在「活動報表」頁面新增活動時，選擇適當的活動類型
3. 系統會自動根據活動類型設定預約開放時間

### 學生
1. 在活動日曆中查看活動
2. 系統會根據活動類型顯示正確的預約開放時間
3. 只有在預約開放時間內才能進行預約

## 注意事項

1. 現有的 English Table 活動不受影響，保持原有的預約時間邏輯
2. 新增的活動類型預約時間會提前更多天開放，請確保活動資訊提前準備
3. 資料庫遷移前請先備份資料
4. 建議在維護時間執行資料庫遷移
