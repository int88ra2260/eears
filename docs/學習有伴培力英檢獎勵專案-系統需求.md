# 學習有伴培力英檢獎勵專案 — 系統需求對照

本文件對照「學習有伴培力英檢獎勵專案」辦法，整理報名與管理系統需支援之功能與規則。

---

## 一、專案規則與系統對應

| 專案規則 | 系統需求 | 目前狀態 |
|----------|----------|----------|
| 限完成報名並實際應考聽說讀寫四項 | 報名時須選擇報考「聽說讀寫」；成績/出席需與報名資料串接 | 已有個人報考項目（LRSW/LR/SW/NON），需可區分「有參與本獎勵專案」 |
| 聽說讀 CEFR B1+、寫作 B1+ | 成績審核或匯入時可判斷等級 | 依後續成績來源（官方/手動）實作 |
| 每團 3～4 人 | 團體報名表單：成員人數 3～4，且每人僅能加入一團 | **已實作**：團體報名流程與資料結構 |
| 每人限參與一組 | 報名/查詢時檢查：該學號是否已在任一團體中 | **待開發** |
| 由一名成員代表填寫學習夥伴報名表單 | 團體報名：代表登入/驗證後，填寫團體名稱 + 勾選/輸入成員（學號等） | **待開發**：學習夥伴報名表單 |
| 團隊名額 50 組為限 | 後台或報名時顯示已報名組數；滿 50 組即不再開放團體報名 | **待開發**：名額計數與開關 |

---

## 二、獎勵計算與系統支援

| 獎勵類型 | 計算邏輯 | 系統需求 |
|----------|----------|----------|
| 出席獎勵 A | 個人四項皆到考 | 需記錄每人聽/說/讀/寫是否到考（或由成績/考務系統提供） |
| 出席獎勵 B | 團隊全員四項皆到考 | 需能列出「團體」及其成員，並彙總每人到考狀態 |
| 個人獎 | 全體中單人表現最佳 | 需可匯出/查詢「有參加本專案」之考生成績，排序取第一名 |
| 團體排名獎 | 團體聽說讀寫平均排名 1～20 | 需：團體列表、每團體成員、每成員四項成績 → 算團體平均 → 排名 |

**結論**：系統需具備「團體」與「團體成員」資料，以及與成績/到考資料的關聯，才能計算上述獎勵。

---

## 三、建議資料結構（團體報名）

### 3.1 新增概念

- **學習夥伴團體（LearningPartnerTeam）**
  - 團體名稱（可選或自動產生）
  - 代表者學號（填表人）
  - 報名時間
  - 狀態（例如：已送出、已額滿、已取消）

- **團體成員（TeamMember）**
  - 團體 ID
  - 學號、姓名、身分證字號（與現有 `EnglishTestRegistration` 一致）
  - 是否為代表（代表即填表人）
  - 每人限一團 → 唯一性：同一學號只能出現在一個團體

### 3.2 與現有報名的關係

- 成員**必須先完成個人培力英檢報名**（現有個人報名流程），再可被加入團體。
- 團體報名表單：代表登入後，輸入或選擇 2～4 位成員（例如學號 + 姓名 + 身分證驗證），系統檢查：
  - 該學號是否已有個別報名（培力英檢）
  - 該學號是否已在其他團體

### 3.3 名額控制

- 全域：已成立之團體數 < 50 時，才開放「學習夥伴報名表單」。
- 可選：後台設定「學習有伴專案」獨立開關與名額上限（例如 50）。

---

## 四、前端流程建議（學習夥伴報名）

1. **入口**：培力英檢報名頁「團體報名」標籤改為可點，導向「學習夥伴報名」流程。
2. **代表驗證**：代表填寫學號、姓名、身分證字號，系統驗證是否已有個別報名。
3. **填寫團體**：團體名稱（選填）、成員 2～4 人（學號 + 姓名 + 身分證），每次新增成員時檢查是否已報名培力英檢、是否已在其他團。
4. **同意與送出**：勾選同意專案辦法與成績使用，送出後寫入團體與成員，並扣減名額。
5. **名額已滿**：若已達 50 組，隱藏或停用團體報名入口，並顯示「團體名額已滿」。

---

## 五、後台管理建議

- **團體列表**：顯示所有學習夥伴團體（代表、成員、報名時間、狀態）。
- **名額與開關**：顯示已報名組數 / 50；可設定「學習有伴專案」開放/關閉。
- **匯出**：團體與成員清單匯出（供成績串接、出席與獎勵計算）。
- **成績/出席匯入**：若成績來源為外部，需能依學號或報名編號匯入「到考」與「各項成績」，以便計算出席獎勵與團體平均排名。

## 七、實作優先順序建議

1. **第一階段**：資料庫與後端  
   - 新增「學習夥伴團體」與「團體成員」資料表與 API（建立團體、新增成員、名額查詢、重複檢查）。
2. **第二階段**：前端團體報名  
   - 開放「團體報名」入口、代表驗證、成員 2～4 人表單、名額 50 與滿額提示。
3. **第三階段**：後台與獎勵  
   - 團體列表、名額與開關、匯出；若需系統內計算獎勵，再串接成績/到考資料與排名邏輯。
