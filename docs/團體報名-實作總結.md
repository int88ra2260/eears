## 三、API 文件

### 學生端 API

#### 1. POST /api/learning-partner/teams
建立團體報名

**Request Body:**
```json
{
  "teamName": "英語學習小組",  // 選填
  "teamSize": 3,  // 3~4
  "members": [
    { "studentId": "B123456789", "name": "王小明" },
    { "studentId": "B987654321", "name": "李小華" },
    { "studentId": "B111222333", "name": "張小美" }
  ]
}
```

**Response (201):**
```json
{
  "message": "團體報名已建立，請所有成員在 24 小時內完成同意",
  "team": {
    "id": 1,
    "teamName": "英語學習小組",
    "teamSize": 3,
    "status": "pending_approval",
    "expiresAt": "2026-01-31T12:00:00.000Z",
    "members": [...]
  }
}
```

**Error Codes:**
- `LP_QUOTA_FULL` - 名額已滿
- `LP_MEMBER_NOT_ELIGIBLE` - 成員不符合資格
- `LP_MEMBER_ALREADY_IN_TEAM` - 成員已在其他團體中
- `LP_INVALID_TEAM_SIZE` - 團隊人數無效
- `LP_MEMBER_COUNT_MISMATCH` - 成員數量不一致

#### 2. GET /api/learning-partner/teams/:teamId
查詢團體狀態

**Response (200):**
```json
{
  "team": {
    "id": 1,
    "teamName": "英語學習小組",
    "teamSize": 3,
    "status": "pending_approval",
    "remainingHours": 23,
    "remainingMinutes": 45,
    "members": [...]
  }
}
```

#### 3. POST /api/learning-partner/approve/prepare
準備同意頁面（二次確認前）

**Request Body:**
```json
{
  "token": "uuid-token"
}
```

#### 4. POST /api/learning-partner/approve/confirm
確認同意

**Request Body:**
```json
{
  "token": "uuid-token"
}
```

#### 5. POST /api/learning-partner/teams/:teamId/resend
重新發送連結

**Request Body:**
```json
{
  "memberId": 1
}
```

### 管理端 API（需 JWT）

#### 6. GET /api/admin/learning-partner/teams
列表查詢

**Query Parameters:**
- `status` - 狀態篩選
- `q` - 搜尋關鍵字（代表者學號、成員學號、teamId）
- `page` - 頁碼（預設 1）
- `limit` - 每頁筆數（預設 20）

#### 7. GET /api/admin/learning-partner/teams/:teamId
詳情查詢

#### 8. POST /api/admin/learning-partner/teams/:teamId/cancel
取消團體

**Request Body:**
```json
{
  "reason": "取消原因"  // 選填
}
```

#### 9. GET /api/admin/learning-partner/export
匯出資料

**Query Parameters:**
- `format` - 格式（csv/json，預設 csv）

## 四、資料庫 Schema

### learning_partner_teams
- `id` (PK, INT)
- `teamName` (VARCHAR(100), nullable)
- `representativeStudentId` (VARCHAR(50))
- `teamSize` (TINYINT, 3~4)
- `status` (ENUM: pending_approval, approved, expired, cancelled)
- `activeFlag` (TINYINT(1), 1=有效, 0=無效)
- `expiresAt` (DATETIME)
- `approvedAt` (DATETIME, nullable)
- `cancelledAt` (DATETIME, nullable)
- `cancelledReason` (TEXT, nullable)
- `createdAt`, `updatedAt`

**索引:**
- `idx_status_createdAt` (status, createdAt)
- `idx_expiresAt` (expiresAt)
- `idx_representativeStudentId` (representativeStudentId)

### learning_partner_team_members
- `id` (PK, INT)
- `teamId` (FK -> learning_partner_teams.id)
- `studentId` (VARCHAR(50))
- `name` (VARCHAR(50))
- `email` (VARCHAR(100))
- `isRepresentative` (BOOLEAN)
- `personalRegistrationId` (FK -> english_test_registrations.id)
- `approvalStatus` (ENUM: pending, approved, expired)
- `activeFlag` (TINYINT(1))
- `approvalToken` (VARCHAR(255), UNIQUE, nullable)
- `approvalTokenExpiresAt` (DATETIME, nullable)
- `approvedAt` (DATETIME, nullable)
- `approvalIp` (VARCHAR(45), nullable)
- `approvalUserAgent` (TEXT, nullable)
- `createdAt`, `updatedAt`

**索引:**
- `idx_teamId` (teamId)
- `idx_approvalToken` (approvalToken, UNIQUE)
- `idx_personalRegistrationId` (personalRegistrationId)
- `idx_studentId_activeFlag` (studentId, activeFlag)

## 五、手動測試步驟

### 1. 資料庫 Migration
```bash
cd reservation-backend
# 執行 migration（需確認 Sequelize CLI 設定）
npx sequelize-cli db:migrate
# 或使用專案自訂的 migration 腳本
```

### 2. 安裝依賴
```bash
cd reservation-backend
npm install uuid  # 如果尚未安裝
```

### 3. 設定環境變數
確認 `.env` 中有：
- `FRONTEND_URL` - 前端網址（用於產生授權連結）
- `BESTEP_GMAIL_USER` / `BESTEP_GMAIL_PASS` - 郵件服務帳號

### 4. 測試流程

#### A. 建立團體報名
1. 確保至少 3~4 位學生已完成個人培力英檢報名（status = 'success'）
2. 訪問 `/register/english-test/group`
3. 選擇團隊人數（2/3/4）
4. 填寫成員資料（學號 + 姓名）
5. 送出

#### B. 同意流程
1. 檢查成員郵件，取得授權連結
2. 點擊連結進入 `/register/english-test/group/approve?token=xxx`
3. 查看二次確認頁面
4. 點擊「確定同意」
5. 確認狀態更新

#### C. 狀態查詢
1. 訪問 `/register/english-test/group/status/:teamId`
2. 查看成員同意狀態
3. 測試重新發送功能

#### D. 管理端
1. 登入管理端
2. 訪問 `/api/admin/learning-partner/teams`
3. 測試列表、詳情、匯出功能

### 5. 定時任務測試
```bash
# 手動執行一次
node reservation-backend/scripts/learningPartnerExpireCron.js
```

## 六、已知限制與注意事項

### 1. 依賴套件
- 需要安裝 `uuid` 套件：`npm install uuid`

### 2. 資料庫約束
- MySQL 不支援 partial unique index，每人限一組的約束在應用層實現（transaction + SELECT FOR UPDATE）
- 需確保 transaction 正確使用，避免 race condition

### 3. 名額控制
- 名額計算基於 `status IN ('pending_approval', 'approved') AND activeFlag = 1`
- 建立團體時使用 transaction + SELECT FOR UPDATE 鎖定，避免超賣

### 4. Token 安全
- Token 為 UUID v4，一次性使用
- 使用後清空 `approvalToken`，防止重用
- 記錄 IP 和 User-Agent

### 5. 郵件服務
- 依賴 `BESTEP_GMAIL_USER` / `BESTEP_GMAIL_PASS` 環境變數
- 若未設定，郵件會記錄日誌但不發送

### 6. 定時任務
- 每 15 分鐘執行一次（可在 server.js 中調整）
- 需確保 server 持續運行

## 七、回滾方案

### 1. Migration 回滾
```bash
npx sequelize-cli db:migrate:undo
# 或使用專案自訂的 rollback 腳本
```

### 2. 功能開關
- 設定 `learning_partner_enabled = false` 可關閉功能
- 前端會檢查此設定，隱藏團體報名入口

### 3. 資料保留
- 回滾 migration 會刪除表，但可先備份資料
- 建議在回滾前匯出資料
