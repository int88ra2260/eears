> 用途：給行政端在試切換當天照表執行。  
> 原則：先檢查、再試切換、再觀察、最後決定是否維持或回復。

---

## B. 試切換前（先確認）

- [ ] 系統可正常登入與操作  
- [ ] 今日資料已完成更新（或已確認不再變動）  
- [ ] 已完成「準備度檢查」且結果為可試切換  
- [ ] 已完成「資料對照」並確認無重大問題  
- [ ] 已確認如有異常可立即回復舊模式  

> 若以上任一未完成，今天**不進行試切換**。

---

## C. 試切換步驟（執行順序）

1. 記錄「切換前」畫面結果  
   - 儀表板摘要  
   - 學生列表  
   - 學生詳情（抽查）
2. 由技術人員協助切換為新讀取模式（v3）  
3. 重新整理頁面並記錄「切換後」畫面結果  
4. 進行差異比對與操作確認  

---

## D. 切換後必看頁面（行政端）

### 1) 英檢儀表板
- [ ] 數字看得懂、無明顯異常跳動  
- [ ] 與切換前相比差異可解釋  

### 2) 學生列表
- [ ] 沒有明顯漏人（尤其在學名冊學生）  
- [ ] 學生達標與等級欄位看起來合理  

### 3) 學生詳情（至少抽查 10 位）
- [ ] 姓名／年級／系所顯示合理  
- [ ] 成績等級（best skills）合理  
- [ ] 考試紀錄顯示合理  

### 4) 學習歷程中心頁面
- [ ] 準備度與狀態區塊可正常顯示  
- [ ] 行政端可理解目前資料來源說明  

---

## E. 異常判斷（行政版）

### 可繼續維持新模式（v3）
- [ ] 數據差異小且可說明  
- [ ] 學生列表沒有重大漏人  
- [ ] 抽查學生詳情無明顯錯誤  
- [ ] 行政端可理解畫面資訊  

### 必須立即回復舊模式（legacy）
- [ ] 主要摘要數字明顯不合理  
- [ ] 學生列表出現明顯漏人  
- [ ] 學生詳情成績資訊錯誤  
- [ ] 畫面頻繁出錯或無法判讀  

---

## F. 回復步驟（若需）

1. 通知技術人員「立即回復舊模式」  
2. 回復後重新檢查：
   - [ ] 儀表板正常  
   - [ ] 學生列表正常  
   - [ ] 學生詳情正常  
3. 記錄回復時間與原因  

- 回復時間：＿＿＿＿＿＿＿＿  
- 回復原因：＿＿＿＿＿＿＿＿
