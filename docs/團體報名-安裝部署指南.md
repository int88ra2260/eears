# 學習有伴團體報名功能 - 安裝部署指南

## 一、前置需求

### 1. 安裝依賴套件

```bash
cd reservation-backend
npm install uuid
```

如果使用 Jest 進行測試，還需要：
```bash
npm install --save-dev supertest
```

### 2. 環境變數設定

在 `reservation-backend/.env` 中確認以下設定：

```env
# 前端網址（用於產生授權連結）
FRONTEND_URL=http://localhost:3000

# 培力英檢郵件服務（如果尚未設定）
BESTEP_GMAIL_USER=your-email@gmail.com
BESTEP_GMAIL_PASS=your-app-password
```

## 二、資料庫 Migration

### 1. 執行 Migration

```bash
cd reservation-backend

# 方法一：使用 Sequelize CLI（如果已設定）
npx sequelize-cli db:migrate

# 方法二：使用專案自訂的 migration 腳本
node scripts/run-migration.js migrations/20260130000000-create-learning-partner-tables.js
```

### 2. 驗證 Migration

確認以下表已建立：
- `learning_partner_teams`
- `learning_partner_team_members`

```sql
SHOW TABLES LIKE 'learning_partner%';
```

### 3. 回滾 Migration（如需要）

```bash
npx sequelize-cli db:migrate:undo
# 或
node scripts/rollback-migration.js migrations/20260130000000-create-learning-partner-tables.js
```

## 三、功能開關設定

### 1. 啟用功能（預設已啟用）

功能預設為啟用狀態，可透過管理端 API 調整：

```bash
# 啟用
curl -X PUT http://localhost:3000/api/settings/learning-partner-enabled \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": true}'

# 設定名額上限（預設 50）
curl -X PUT http://localhost:3000/api/settings/learning-partner-quota \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quota": 50}'
```

### 2. 關閉功能

```bash
curl -X PUT http://localhost:3000/api/settings/learning-partner-enabled \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

## 四、啟動服務

### 1. 後端服務

```bash
cd reservation-backend
npm start
# 或開發模式
npm run dev
```

確認日誌中出現：
```
⏰ 學習有伴過期檢查定時任務已啟動（每 15 分鐘執行一次）
```

### 2. 前端服務

```bash
cd reservation-frontend
npm start
```

## 五、功能測試

### 1. 基本流程測試

1. **建立團體報名**
   - 訪問 `http://localhost:3000/register/english-test/group`
   - 選擇團隊人數（2/3/4）
   - 填寫成員資料（需確保這些學生已完成個人報名且 status = 'success'）
   - 送出

2. **同意流程**
   - 檢查成員郵件，取得授權連結
   - 點擊連結進入同意頁面
   - 確認資訊後點擊「確定同意」

3. **狀態查詢**
   - 訪問 `http://localhost:3000/register/english-test/group/status/:teamId`
   - 查看成員同意狀態

### 2. API 測試

使用 Postman 或 curl 測試 API：

```bash
# 建立團體
curl -X POST http://localhost:3000/api/learning-partner/teams \
  -H "Content-Type: application/json" \
  -d '{
    "teamSize": 2,
    "members": [
      {"studentId": "B123456789", "name": "王小明"},
      {"studentId": "B987654321", "name": "李小華"}
    ]
  }'

# 查詢狀態
curl http://localhost:3000/api/learning-partner/teams/1
```

### 3. 定時任務測試

手動執行一次過期檢查：

```bash
node reservation-backend/scripts/learningPartnerExpireCron.js
```

## 六、常見問題

### 1. Migration 失敗

**問題：** Migration 執行時出現錯誤

**解決方案：**
- 檢查資料庫連線設定
- 確認 MySQL 版本（建議 5.7+）
- 檢查是否有權限建立表

### 2. 郵件無法發送

**問題：** 團體報名建立後，成員未收到郵件

**解決方案：**
- 檢查 `BESTEP_GMAIL_USER` 和 `BESTEP_GMAIL_PASS` 是否正確設定
- 確認 Gmail 應用程式密碼已啟用
- 檢查後端日誌中的郵件錯誤訊息

### 3. Token 無效

**問題：** 點擊授權連結時顯示「無效的授權連結」

**解決方案：**
- 確認 `FRONTEND_URL` 設定正確
- 檢查 token 是否已過期（24 小時）
- 確認 token 是否已被使用

### 4. 名額檢查失敗

**問題：** 建立團體時顯示「名額已滿」，但實際未滿

**解決方案：**
- 檢查 `learning_partner_quota` 設定值
- 確認資料庫中有效團體數量計算正確
- 檢查是否有 expired/cancelled 的團體未正確釋出名額

## 七、監控與維護

### 1. 日誌監控

關注以下日誌訊息：
- `✅ Learning Partner tables created successfully` - Migration 成功
- `📧 Email sent successfully` - 郵件發送成功
- `⏰ 學習有伴過期檢查定時任務已啟動` - 定時任務啟動
- `✅ 已處理 X 個過期團體` - 定時任務執行結果

### 2. 資料庫維護

定期檢查：
- 過期團體數量（`status = 'expired'`）
- 待同意團體數量（`status = 'pending_approval'`）
- 已完成團體數量（`status = 'approved'`）

### 3. 效能優化

如果團體數量很大，考慮：
- 為 `expiresAt` 欄位建立索引（已建立）
- 優化定時任務查詢（使用批次處理）
- 考慮使用 Redis 快取名額計數

## 八、安全注意事項

1. **Token 安全**
   - Token 為 UUID v4，不可猜測
   - 一次性使用，使用後立即清空
   - 記錄 IP 和 User-Agent，便於追蹤

2. **交易一致性**
   - 所有關鍵操作使用 transaction
   - 使用 SELECT FOR UPDATE 鎖定，避免 race condition

3. **權限控制**
   - 管理端 API 需 JWT 認證
   - 取消團體功能僅限管理員

## 九、回滾步驟

如果需回滾功能：

1. **關閉功能開關**
   ```bash
   curl -X PUT http://localhost:3000/api/settings/learning-partner-enabled \
     -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"enabled": false}'
   ```

2. **備份資料**（如需要）
   ```sql
   -- 匯出資料
   SELECT * FROM learning_partner_teams INTO OUTFILE '/tmp/teams_backup.csv';
   SELECT * FROM learning_partner_team_members INTO OUTFILE '/tmp/members_backup.csv';
   ```

3. **回滾 Migration**
   ```bash
   npx sequelize-cli db:migrate:undo
   ```

4. **移除路由**（可選）
   - 從 `server.js` 中移除 `learningPartnerRouter`
   - 從前端 `App.js` 中移除相關路由
