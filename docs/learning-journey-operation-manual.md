# Learning Journey 營運操作手冊（行政版）

本手冊提供非開發人員日常使用 Learning Journey 的最小操作流程，避免誤判資料或誤操作。

## 1. 如何查看 KPI

1. 進入 `/admin/learning-journey`
2. 維持「營運模式（Operation Mode）」
3. 輸入學期後按「正式版」
4. 主要看四個指標：
   - 達標率
   - 英檢報名率
   - 活動參與率
   - 風險學生數

若出現「資料可能過舊，請聯絡管理員」，請先停止解讀指標並通知管理員。

## 2. 如何查看學生

1. 在同頁上方輸入學號
2. 按「載入」
3. 查看學生基本資料、成績與活動摘要

若資料看起來明顯不合理（例如學號正確但完全空白），請記錄學號與學期後通知工程師。

## 3. 什麼是 Risk Students

Risk Students 是系統依規則標記的高風險名單，常見原因：
- 無任何英檢紀錄
- 多次未達標
- 活動參與偏低
- 已報名但查無應試/出席紀錄

此名單用於優先關懷，不代表最終行政裁定。

## 4. 什麼時候需要找工程師

請在以下情況立即通知工程師：
- 畫面提示資料過舊（stale）
- 主要 KPI 與既有認知差距很大
- 同學期資料忽然大幅變少
- 風險名單大量異常（例如全部學生都被標風險）
- 頁面一直出現錯誤訊息（紅色警示）

## 5. 不應該做的事情

- 不要在不清楚影響時執行 sync
- 不要自行切換 read model 旗標
- 不要只看單一頁面就下結論
- 不要忽略「資料過舊」或警示訊息

## 6. 建議日常流程（簡版）

1. 開啟營運模式
2. 先看 KPI 與警示
3. 再看風險學生
4. 抽查關鍵學生資料
5. 有異常就回報（附學期與學號）
