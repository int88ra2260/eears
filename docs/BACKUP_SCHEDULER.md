# EEARS Level 1 備份：Windows 工作排程器設定

目標：每天 **03:00** 自動執行 `scripts\backup-db.bat`，並在**未登入**時仍可執行。

## 建議帳號

- 使用具備下列權限的 **Windows 本機或網域帳號**：
  - 讀取 `reservation-backend\.env`
  - 寫入 `D:\EEARS_backup\`（`local`、`logs`）
  - 存取 MySQL（`mysqldump` 所需之 DB 帳號已由 `.env` 提供）
  - 若使用 USB：需能寫入 `E:\EEARS_backup\`（隨身碟插入時）
  - 若使用 rclone：需能讀取該帳號下設定好的 rclone 設定檔（通常在使用者設定檔目錄）

## 建立排程工作（逐步）

1. 開啟 **工作排程器**（`taskschd.msc`）。
2. **建立工作…**（勿只用「建立基本工作」，以便勾選「即使未登入也執行」）。
3. **一般**：
   - **名稱**：例如 `EEARS_DB_Backup_Level1`。
   - 勾選 **以最高權限執行**（若路徑或服務需要；否則可視環境關閉）。
   - 選擇 **不論使用者登入與否均執行**（對應「即使未登入也執行」）。
   - 設定 **使用者或群組** 為上述備份專用帳號（或 `SYSTEM`；若選 `SYSTEM`，請確認 `.env` 與 rclone 設定路徑對該內容可讀）。
4. **觸發程序** → **新增**：
   - **每日**。
   - **開始時間**：`03:00:00`。
   - 已啟用：是。
5. **動作** → **新增**：
   - **動作**：啟動程式。
   - **程式或指令碼**：`cmd.exe`
   - **新增引數**（請將路徑改為您伺服器上實際的 EEARS 專案根目錄）：

     ```text
     /c "F:\EEARS_backup_20251211\scripts\backup-db.bat"
     ```

     若路徑含空格，請維持外層雙引號。
   - **開始於**（選填但建議）：`F:\EEARS_backup_20251211`（專案根目錄，有助相對路徑解析）。
6. **條件**／**設定**：依機房政策調整（例如僅在 AC 供電時執行筆電策略等）。
7. 若使用「不論使用者登入與否均執行」，系統可能要求輸入該帳號密碼以儲存認證。

## 驗證

1. 在工作上按右鍵 → **執行**。
2. 檢查 `D:\EEARS_backup\local\` 是否出現 `eears_YYYY-MM-DD_HH-mm.sql.gz`。
3. 檢查 `D:\EEARS_backup\logs\backup.log` 是否新增紀錄。

## 常見問題

- **工作顯示成功但無備份檔**：檢查「開始於」目錄、`.env` 路徑、`EEARS_ENV_FILE` 是否於排程環境中可見。
- **mysqldump 找不到**：在該排程帳號的 PATH 中加入 MySQL `bin`，或使用「系統」環境變數 PATH。
- **rclone 失敗**：以同一帳號開啟命令提示字元手動執行 `backup-db.bat` 測試；OAuth 常綁定使用者設定檔。
