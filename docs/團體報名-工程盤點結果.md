# 團體報名功能 - 工程盤點結果

## 一、技術棧確認

### 後端
- **ORM**: Sequelize
- **資料庫**: MySQL
- **Migration 機制**: Sequelize migration（JS 格式）
- **郵件服務**: nodemailer + emailQueue.js（非阻塞佇列）
- **認證**: JWT（authMiddleware, adminMiddleware）
- **Feature Flags**: Settings 表（key-value，支援 valueBool）

### 前端
- **框架**: React + React Router
- **路由配置**: App.js 中使用 Routes/Route
- **UI 風格**: Bootstrap + 自定義樣式（藍白學術風格）

## 二、現有系統關鍵資訊

### EnglishTestRegistration 模型
- **Status 值**: `pending`, `approved`, `revision`, `success`, `failed`
- **確認**: `status = 'success'` 存在，用於判斷「報名成功」
- **唯一約束**: `studentId` 為 unique
- **Email 欄位**: `email` (STRING(100))

### 郵件系統
- **培力英檢專用帳號**: `BESTEP_GMAIL_USER` / `BESTEP_GMAIL_PASS`
- **郵件模板**: `config/email.js` 中的 `emailTemplates` 物件
- **發送方式**: `emailQueue.enqueue(template, data)` 或直接 `sendEmail(template, data)`

### Settings 系統
- **表結構**: `key` (unique), `value` (TEXT), `valueBool` (BOOLEAN)
- **現有設定**: `english_test_registration_enabled`（培力英檢報名開關）
- **API**: `/api/settings/:key` (GET/PUT)

### 路由結構
- **後端**: `/api/learning-partner/*`（需新增）
- **前端**: `/register/english-test/group`（需新增）

## 四、關鍵設計決策

### 資料庫約束
- **每人限一組**: 使用 `activeFlag` 輔助欄位 + `UNIQUE(studentId, activeFlag)` 實現 partial unique index
- **名額控制**: 在建立團體時使用 transaction + SELECT FOR UPDATE 鎖定，確保不會超賣

### Token 機制
- **格式**: UUID v4
- **儲存**: `approvalToken` (VARCHAR(255) UNIQUE)
- **時效**: `approvalTokenExpiresAt` (DATETIME = createdAt + 24h)
- **一次性**: 使用後清空 `approvalToken` 或標記 `approvalStatus = 'approved'`

### 名額管理
- **佔用時機**: 送出時立即佔用（`status = 'pending_approval'` 或 `approved'`）
- **釋放時機**: `expired` 或 `cancelled` 時釋放（`activeFlag = 0`）
- **計算方式**: `COUNT(*) WHERE status IN ('pending_approval', 'approved') AND activeFlag = 1`
