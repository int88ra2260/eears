# Pull Request 模板

## 📋 變更說明

### 變更類型
- [ ] 安全性強化
- [ ] 程式碼重構
- [ ] 新功能
- [ ] 錯誤修復
- [ ] 文件更新
- [ ] 測試

### 變更摘要
簡要描述此 PR 的主要變更內容。

### 詳細說明
詳細說明變更的原因、實施方式和影響範圍。

## 🔍 風險分析

### 潛在風險
- [ ] 可能影響現有功能
- [ ] 可能影響效能
- [ ] 可能影響安全性
- [ ] 可能影響部署流程
- [ ] 可能影響資料庫

### 風險等級
- [ ] 低風險 (不影響對外介面)
- [ ] 中風險 (可能影響部分功能)
- [ ] 高風險 (可能影響核心功能)

### 緩解措施
說明如何降低或避免潛在風險。

## 🔄 回滾方式

### 自動回滾觸發條件
- [ ] 測試失敗率 > 10%
- [ ] 系統可用性 < 95%
- [ ] 安全掃描發現新漏洞
- [ ] 效能下降 > 20%

### 手動回滾步驟
1. 停止新版本部署
2. 回滾到上一個穩定版本
3. 分析問題原因
4. 修復後重新部署

### 回滾命令
```bash
# 回滾到上一個版本
git checkout previous-stable-tag
npm install
npm start
```
