# 審核進度計算邏輯 - 篩選條件分析

## 📋 篩選條件列表

### 1. 狀態篩選 (status)
- `all` - 全部
- `pending` - 待審核
- `approved` - 已通過
- `rejected` - 已拒絕

### 2. 搜尋條件 (search)
- 學號 (studentId)
- 姓名 (name)
- Email (email)

### 3. 日期範圍 (dateFrom, dateTo)
- 報名時間範圍

### 4. 測驗類型 (examTypes[])
- `LRSW` - 四項全考
- `LR` - 聽讀
- `SW` - 說寫
- `NON` - 不報考
- 可複選

### 5. 特殊身分
- `isLowIncome` - 中低收入戶（否/中低收入戶/低收入戶）
- `hasDisabilityCard` - 身心障礙手冊（是/否）

## 🧮 審核進度計算邏輯

### 基本公式
```
審核進度 = (已通過數量 / 總數量) × 100%
```

### 各種篩選條件下的分子分母

#### 情況 1：無任何篩選條件
- **分母（總數）**：所有報名記錄數
- **分子（已通過）**：所有狀態為 `approved` 的記錄數
- **計算**：`approved / total`

#### 情況 2：僅狀態篩選
- **狀態 = `all`**：
  - 分母：所有記錄數
  - 分子：所有 `approved` 記錄數
- **狀態 = `pending`**：
  - 分母：所有 `pending` 記錄數
  - 分子：0（因為已篩選為 pending，沒有 approved）
  - **特殊處理**：此情況下進度條應顯示 0% 或隱藏
- **狀態 = `approved`**：
  - 分母：所有 `approved` 記錄數
  - 分子：所有 `approved` 記錄數
  - **特殊處理**：此情況下進度條應顯示 100%
- **狀態 = `rejected`**：
  - 分母：所有 `rejected` 記錄數
  - 分子：0
  - **特殊處理**：此情況下進度條應顯示 0% 或隱藏

#### 情況 3：日期範圍篩選
- **分母**：在日期範圍內的記錄數
- **分子**：在日期範圍內且狀態為 `approved` 的記錄數

#### 情況 4：測驗類型篩選
- **分母**：符合測驗類型的記錄數
- **分子**：符合測驗類型且狀態為 `approved` 的記錄數

#### 情況 5：特殊身分篩選
- **分母**：符合特殊身分條件的記錄數
- **分子**：符合特殊身分條件且狀態為 `approved` 的記錄數

#### 情況 6：搜尋條件
- **分母**：符合搜尋條件的記錄數
- **分子**：符合搜尋條件且狀態為 `approved` 的記錄數

#### 情況 7：多條件組合
- **分母**：符合所有篩選條件的記錄數
- **分子**：符合所有篩選條件且狀態為 `approved` 的記錄數

## ⚠️ 特殊情況處理

### 1. 狀態篩選為 `pending` 或 `rejected`
- **建議**：不顯示進度條，或顯示「無需審核」提示
- **原因**：這些狀態下沒有已通過的記錄

### 2. 狀態篩選為 `approved`
- **建議**：顯示 100% 進度條，或顯示「全部已通過」提示
- **原因**：所有記錄都是已通過狀態

### 3. 分母為 0
- **建議**：不顯示進度條，或顯示「無資料」提示
- **原因**：無法計算百分比

### 4. 僅有日期範圍篩選
- **計算**：在該日期範圍內的所有記錄中，已通過的比例
- **用途**：查看特定時間段的審核進度

### 5. 僅有測驗類型篩選
- **計算**：該測驗類型的所有記錄中，已通過的比例
- **用途**：查看特定測驗類型的審核進度

## 🔄 實作建議

### 後端 API 修改
1. 統計資訊應根據 `where` 條件計算（而非所有資料）
2. 分別計算：
   - `total`：符合篩選條件的總數
   - `approved`：符合篩選條件且狀態為 `approved` 的數量
   - `pending`：符合篩選條件且狀態為 `pending` 的數量
   - `rejected`：符合篩選條件且狀態為 `rejected` 的數量

### 前端組件修改
1. 使用篩選後的統計資訊計算進度
2. 特殊情況處理：
   - 狀態篩選為 `pending` 或 `rejected`：不顯示或顯示 0%
   - 狀態篩選為 `approved`：顯示 100%
   - 分母為 0：不顯示或顯示提示

## 📝 範例計算

### 範例 1：無篩選
- 總記錄：100
- 已通過：60
- 待審核：30
- 已拒絕：10
- **進度**：60 / 100 = 60%

### 範例 2：日期範圍篩選（2024-01-01 至 2024-01-31）
- 範圍內總記錄：50
- 範圍內已通過：35
- **進度**：35 / 50 = 70%

### 範例 3：測驗類型篩選（聽讀）
- 聽讀總記錄：40
- 聽讀已通過：25
- **進度**：25 / 40 = 62.5%

### 範例 4：狀態篩選（待審核）
- 待審核總記錄：30
- 待審核已通過：0
- **進度**：0 / 30 = 0%（或隱藏進度條）

### 範例 5：多條件組合（日期範圍 + 測驗類型）
- 符合條件總記錄：20
- 符合條件已通過：15
- **進度**：15 / 20 = 75%
